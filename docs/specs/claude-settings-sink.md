# Claude Code 설정 동기화 도구

## 목표

로컬에서만 관리되는 Claude Code 설정(settings.json, skills, commands 등)을 GitHub 원격 저장소와 동기화하여 여러 환경에서 일관된 설정을 유지한다.

## 상세

### 개발 스택

- 언어: Rust
- 인터페이스: CLI
- GitHub 인증: gh CLI 재사용 (`gh auth status` 활용)

### 동기화 대상

- 화이트리스트 기반으로 명시적으로 등록한 파일만 동기화
- 두 가지 방식 지원:
  - 로컬 설정 파일에 동기화할 경로 목록 명시
  - 원격 저장소에 존재하는 파일을 기준으로 자동 동기화

### 원격 저장소

- GitHub private repo 사용
- 사용자가 초기 설정 시 repo URL을 직접 입력
- 디렉토리 구조는 `~/.claude`와 동일하게 유지

### 충돌 해결

- 로컬과 원격 내용이 다를 때 항상 diff를 보여주고 사용자가 선택
- 선택지: 로컬 유지 / 원격으로 덮어쓰기 / 병합 취소

### 백업

- pull 등 로컬 파일이 변경되는 작업 전에 자동 백업 생성
- 백업 위치: `~/.claude.backup/` 또는 유사한 경로

### CLI 명령어 구조

```
clync push              # 로컬 → 원격
clync pull              # 원격 → 로컬
clync diff              # 로컬/원격 비교
clync status            # 동기화 상태 확인
clync config            # 설정 관리 (repo URL, 화이트리스트 등)
```

### 옵션

- `--dry-run`: 실제 변경 없이 예상 결과만 미리보기
- `--force`: 확인 프롬프트 생략 (주의 필요)

## 경계

- ✅ 항상:
  - 화이트리스트에 등록된 파일만 동기화
  - 변경 전 로컬 파일 자동 백업
  - 충돌 시 diff 표시 후 사용자 확인
  - gh CLI 인증 상태 확인 후 작업 진행

- ⚠️ 먼저:
  - 로컬/원격 내용이 다를 때 어느 쪽을 선택할지 확인
  - 새 파일을 원격에 추가할 때 확인

- 🚫 절대:
  - 화이트리스트에 없는 파일 동기화
  - 확인 없이 로컬 파일 덮어쓰기
  - 암호화 (평문 저장, private repo 전제)
  - Claude Code의 모든 설정을 자동으로 동기화

## 검증

- 핵심 로직 단위 테스트:
  - diff 계산 로직
  - 화이트리스트 필터링 로직
  - 파일 경로 매칭 로직
- 수동 검증:
  - 실제 GitHub repo와 push/pull 테스트
  - dry-run 모드로 예상 결과 확인
